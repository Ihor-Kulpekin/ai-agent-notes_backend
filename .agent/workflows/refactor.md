# Refactoring Pipeline

При виконанні рефакторингу слідуйте наступному алгоритму:

1. **Аналіз на 'code smells'**:
   - Великі класи чи методи.
   - Надмірне використання примітивів (Primitive Obsession).
   - "Дублювання коду" або глибока вкладеність циклів/умов.
2. **Пропозиція патернів**:
   - Використання патернів **Strategy**, **Factory**, або **Command** для зменшення зв'язаності.
3. **TypeScript Оптимізація**:
   - Поліпшити роботу з типізацією (позбутися `any`, використати Generics там, де доцільно, забезпечити строгу типізацію).
