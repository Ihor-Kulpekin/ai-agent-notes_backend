# Clean Code
- **DRY (Don't Repeat Yourself)**: Уникайте дублювання коду.
- **KISS (Keep It Simple, Stupid)**: Код має бути максимально простим.
- **YAGNI (You Aren't Gonna Need It)**: Не пишіть код для функцій, які знадобляться "можливо, колись".
- **Single Responsibility**: Один клас/модуль/функція = одна відповідальність.
- **Ніяких magic strings/numbers**: Усі літерали повинні бути винесені в константи або `enum`.
- **Intention-revealing імена**: Імена змінних та методів мають чітко розкривати свій намір (наприклад, `getUserById` замість `getData`).
