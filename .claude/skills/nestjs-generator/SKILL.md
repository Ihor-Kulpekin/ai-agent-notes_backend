# NestJS Architecture
- **Модульна структура**: Жорстке дотримання модульності NestJS.
- **Dependency Injection**: Використання DI-контейнера для всіх залежностей.
- **Guards та Interceptors**: Використовувати для перевірок доступу та трансформації даних відповідно.
- **Чіткі межі шарів**:
    - **Controller**: Тільки роутинг та DTO валідація. Жодної бізнес-логіки.
    - **Service**: Виключно бізнес-логіка додатку.
    - **Repository**: Тільки робота з базою даних або OpenSearch.
