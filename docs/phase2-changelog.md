# Phase 2 Changelog: Memory & Database (RAG Layer)

## 1. Рефакторинг OpenSearchRepository

### Efficient k-NN Pre-filtering
- **Змінений метод:** `knnSearchWithFilter`
- **Суть змін:** Замість Post-filtering (де фільтр застосовувався до масиву результатів k-NN), реалізовано Pre-filtering. Парамтер `filter` тепер знаходиться безпосередньо всередині об'єкта `knn.embedding`. Це гарантує, що k-NN пошук оцінює метрики відстані тільки серед тих документів, які вже відповідають заданим фільтрам (наприклад, належать конкретному `userId`), усуваючи проблему "нульових результатів".

### Hybrid Search & Reciprocal Rank Fusion (RRF)
- **Змінений метод:** `hybridSearch`
- **Суть змін:** Старий підхід `bool.should` із ручним зазначенням `boost: 2.0` видалено через несумісність масштабів оцінок BM25 та Cosine Similarity.
Замість цього реалізовано ручний **RRF (Reciprocal Rank Fusion)**:
1. Виконуються два окремих запити: один для повнотекстового пошуку (BM25) і один для семантичного (kNN) за допомогою `Promise.all`.
2. Результати комбінуються в пам'яті за формулою `score = 1.0 / (60 + rank)`.
3. Документи сортуються за новим `rrfScore`, що забезпечує математично коректне поєднання релевантності.

## 2. Token Window Protection (Hard Cap)
- **Файл:** `src/services/memory/short-term.memory.ts`
- **Змінений метод:** `getActiveWindow`
- **Суть змін:** Додано захисний механізм. Якщо поточне контекстне вікно (`window`) перевищує ліміт (`stm.maxWindowTokens`, стандартно 3000 токенів), старі повідомлення примусово видаляються через цикл `window.shift()`, поки розмір вікна не стане безпечним. Це діє як жорсткий запобіжник (Hard Cap) від помилки переповнення токенів, яка могла виникати до завершення фонової сумаризації.

## 3. Перевірка типів
- Код успішно компілюється TypeScript у Strict Mode (`npx tsc --noEmit`).
- Усі нові структури даних (Map для RRF скорів, openSearchClient) надійно типізовані, використання `any` уникнуто.
- NestJS DI архітектура і LangGraph Checkpointer (`checkpointer.module.ts`) збережені та функціонують згідно з правилами Фази 1.
