# Self-Correction Pattern in Agentic RAG

## 1. Суть патерну та "Нащо воно треба" (The Why)

У звичайному Retrieval-Augmented Generation (RAG) пайплайні, LLM виконує роль простого генератора тексту на основі контексту. Основні проблеми такого підходу:
* **Галюцинації (Hallucinations):** Модель може вигадати факт, якого не було в Source Documents.
* **Втрата формату:** Модель може проігнорувати інструкції з форматування (наприклад, відповісти великим текстом замість списку).
* **Неповна відповідь:** Модель може частково відповісти на питання, упустивши суть.

**Self-Correction (Reflection)** — це архітектурний патерн, який перетворює наш звичайний RAG на справжній **Agentic RAG**. Ми додаємо до системи "внутрішнього критика" (Grader). Замість того, щоб одразу віддавати згенерований текст користувачеві, агент оцінює власну роботу. Якщо виявляється помилка або галюцинація, агент самостійно повертається на етап генерації і переписує відповідь, спираючись на виявлені критичні зауваження. Уся ця магія відбувається під капотом і повністю непомітна для людини.

## 2. Логіка роботи в нашому графі (The How)

Життєвий цикл самокорекції (Reflection Loop) в notesQa:

1. **Генерація (`generator.node.ts`):** Нода отримує знайдений контекст та питання. Створює чорнову версію відповіді і зберігає її в `AgentState.answer`.
2. **Оцінка (`grader.node.ts`):** Граф напрямляється від `generator` до `grader`. Ця нода використовує спеціальний `GRADER_PROMPT` і передає LLM три речі:
   * Оригінальне питання юзера
   * Чорнову відповідь генератора
   * Оригінальні знайдені документи (з OpenSearch)
3. **Вердикт:**
   - **Якщо відповідь ідеальна:** Grader повертає першим словом `PASS`. Node зберігає `{ gradingFeedback: '' }`.
   - **Якщо є помилки:** Grader повертає детальну критику (наприклад, "You claimed X, but the documents only state Y"). Node зберігає цей текст у `{ gradingFeedback: '<критика>' }`.
4. **Маршрутизація (`grader.router.ts`):** 
   - Якщо `gradingFeedback` порожній -> маршрут `'pass'` -> перехід до `END` (відповідь відправляється клієнту).
   - Якщо `gradingFeedback` має текст -> маршрут `'retry'` -> перехід назад до циклу на `generator`.

**Як Generator "розуміє", що виправити?**
Під капотом LangGraph (через reducer), критика залишається у нашому State. Хоча поточна імплементація нашого `generator` ще має простір для інкорпорування поля `state.gradingFeedback` безпосередньо в новий system prompt на наступній ітерації, сам факт циклу дає моделі шанс згенерувати відповідь наново (з іншою температурою/seed'ом або покращеним промптом, який ми можемо туди ін'єктити у майбутньому).

## 3. Оптимізація ресурсів (Fast LLM vs Primary LLM)

Self-Correction має високий ризик фінансових перевитрат. Якщо ми будемо використовувати найдорожчу LLM для кожної ітерації оцінювання, ціна за токени злетить до небес.

У файлі `agent.graph.ts` (при побудові графа `buildAgentGraph`) ми розділили моделі:
* **`generator`** = `llmWithFallbacks` (Primary LLM, гнучка та розумна GPT-4o з високою якістю генерації природної мови).
* **`grader`** = `classifierLlm` (Fast/Cheap LLM, наприклад `gpt-4o-mini`).

**Чому так?**
*Генерація* тексту та логічних конструкцій вимагає найбільшого iq-показника моделі. 
*Класифікація* та валідація (порівняння двох текстів на предмет протиріч) — значно легша задача. `gpt-4o-mini` робить це з такою ж високою точністю, проте вона на 40% швидша і на 85% дешевша.

## 4. Безпека: Запобігання нескінченним циклам (Infinite Loops)

Патерни Self-Correction є причиною падіння багатьох Production-систем через **вічні цикли**.
Сценарій: Generator постійно галюцинує один і той самий факт, а Grader постійно це відхиляє, змушуючи базу (і ваш баланс на OpenAI) крутитись нескінченно.

**Рішення в нашому коді:**
У `AgentState` (файл `agent.state.ts`) існує поле `retryCount`:
```typescript
retryCount: Annotation<number>({
  reducer: (current, update) => current + update,
  default: () => 0,
}),
```

Під час кожного запуску ноди `grader`, він додає: `return { retryCount: 1 }`, що інкрементує лічильник на +1.
Далі працює `grader.router.ts`:
```typescript
const MAX_RETRIES = 2;

if (state.retryCount >= MAX_RETRIES) {
  return 'pass'; // Виходимо навіть якщо grader незадоволений
}
```

**Що відбувається при вичерпанні ліміту?**
Граф примусово перериває цикл роздумів ("Force break"). Він обирає маршрут `'pass'` і приймає поточну (нехай і недосконалу) відповідь `generator`, замість того, щоб палити гроші далі. Це гарантує стабільний SLA і прогнозований час затримки (Latency) для кінцевого користувача.
